## 2026-05-17 - DOM-based XSS in Webpack HtmlWebpackPlugin template
**Vulnerability:** The `HtmlWebpackPlugin` template in `webpack.config.js` was reading the `widgetName` parameter directly from the URL query string (`window.location.search`) and injecting it into the `src` attribute of a dynamically created `<script>` element without any validation. It was also directly appending to `document.body.innerHTML`. This allows DOM-based Cross-Site Scripting (XSS), where an attacker could provide a malicious URL to execute arbitrary JavaScript within the plugin's context.
**Learning:** Query parameters should always be treated as untrusted input. Directly injecting them into potentially dangerous contexts (like script sources or `innerHTML`) without validation or sanitization is a significant security risk.
**Prevention:** Implement an allowlist approach. Validate the `widgetName` against a list of known valid widget entry points generated at build time. Only load the script if the name matches a valid entry in the allowlist. Additionally, use `textContent` instead of `innerHTML` when displaying messages to prevent HTML injection.
